# marzban_disable_inbound.py

Интерактивный CLI-скрипт для **массового отключения выбранных inbound** у всех пользователей панели **Marzban** через её REST API.
Пользователь при этом не удаляется и не блокируется — указанные inbound-теги просто исчезают из поля `inbounds`, поэтому ссылки/конфиги перестают включать эти протоколы.

## Возможности

- Авторизуется в Marzban через `username` / `password` (OAuth2 password flow на `/api/admin/token`).
- Загружает список всех inbound с `/api/inbounds` и отображает их нумерованным списком.
- Спрашивает, какие inbound отключить (несколько — через запятую).
- Получает всех пользователей через `/api/users` с **пагинацией**, **повторными попытками** и **прогрессом**.
- Для каждого пользователя выполняет `PUT /api/user/{username}` с обновлённым массивом `inbounds`.
- Пользователи, у которых выбранных inbound уже не было, помечаются как `unchanged` — лишние запросы не отправляются.
- В конце выводится итог: обновлено / без изменений / с ошибками.

## Требования

- Python **3.10+** (используются type-аннотации `dict[...]`, `list[...]`).
- Пакет `requests`.

Установка зависимости:

```
pip install requests
```

## Запуск

```
python marzban_disable_inbound.py
```

Скрипт спросит:

1. **Marzban URL** — корень API панели. Например, `https://api.example.com`.
   - Важно: это **не** UI-путь с `DASHBOARD_PATH` (секретный случайный префикс используется только для веб-интерфейса).
   - API всегда располагается в корне домена: `https://<host>/api/...`.
2. **Admin username** — логин администратора Marzban.
3. **Admin password** — пароль (ввод скрыт).

Далее — выберите номера inbound, подтвердите `yes`, и скрипт применит изменения ко всем пользователям.

## Используемые API

| Метод | Endpoint                  | Назначение                          |
|-------|---------------------------|-------------------------------------|
| POST  | `/api/admin/token`        | Получить JWT (form-data)            |
| GET   | `/api/inbounds`           | Список всех inbound по протоколам   |
| GET   | `/api/users?offset&limit` | Список пользователей с пагинацией   |
| PUT   | `/api/user/{username}`    | Обновить `inbounds` пользователя    |

Все запросы передаются с заголовком `Authorization: Bearer <token>`.

## Устойчивость к сетевым проблемам

Скрипт рассчитан на работу через Cloudflare / WAF / медленный канал между вами и Marzban:

- Используется **`requests.Session`** — TLS-хендшейк переиспользуется.
- Подключён **retry-адаптер** из `urllib3` (5 попыток, экспоненциальный backoff 2/4/8/16 сек) для кодов `429, 500, 502, 503, 504` и сетевых сбоев.
- Раздельные таймауты `(connect, read)`:
  - login / inbounds / PUT — `(10, 30)`
  - `/api/users` — `(15, 120)` (самый тяжёлый endpoint).
- Размер страницы `/api/users` — **25** (не 100), чтобы CDN не обрывал длинные ответы.
- На каждую страницу предусмотрено до 3 дополнительных ручных повторов с паузами.

Если connect timeout на `/api/users` всё равно возникает — скорее всего, это фильтр WAF/Cloudflare. Варианты обхода:

- Запустить скрипт прямо на сервере с Marzban (через SSH).
- Настроить SSH-туннель и обращаться через `http://127.0.0.1:<port>`.
- Добавить `/api/users` в белый список Cloudflare/WAF.

## Что именно меняется у пользователя
Поле `inbounds` в Marzban — это словарь `{ protocol: [tag1, tag2, ...] }`. Скрипт:

1. Берёт текущий `user.inbounds`.
2. Для каждого протокола фильтрует список тегов, удаляя выбранные.
3. Отправляет `PUT /api/user/{username}` с телом `{ "inbounds": <новый словарь> }`.

Остальные поля пользователя (`proxies`, `expire`, `data_limit`, статус и т.д.) **не затрагиваются** — Marzban принимает частичные обновления.
Если у пользователя выбранных inbound уже нет, скрипт пропускает его без запроса.


## Ограничения / известные проблемы
- **Нет dry-run.** Невозможно заранее увидеть, кого затронут изменения.
- **Нет резервной копии.** Скрипт не сохраняет старые значения `inbounds` перед записью.
- **Прерывание (Ctrl+C) в середине работы** — часть пользователей уже будет изменена, часть — нет. Безопасно перезапустить: изменённые пользователи будут помечены как `unchanged`.
- **Совместимость**: протестировано на текущей версии Marzban API (`/api/admin/token` + `UserResponse.inbounds: Dict[str, List[str]]`). На старых версиях панели структура полей может отличаться.
